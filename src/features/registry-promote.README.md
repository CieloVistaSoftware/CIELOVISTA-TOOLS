# registry-promote

Register a folder as a CieloVista product. One-click alternative to hand-editing `project-registry.json`.

## Command

`cvs.registry.promote` — "Promote Folder to Product"

## How it's invoked

- Right-click any folder in the VS Code Explorer → **Promote Folder to Product**
- Command palette → runs against the workspace root, or opens a folder picker
- Programmatically via `vscode.commands.executeCommand('cvs.registry.promote', uri)`

## What it does

1. Resolves the target folder — from the context-menu URI, the single workspace root, or an open-dialog pick.
2. Prompts for project **name** (defaults to folder basename), **type** (from a fixed list: vscode-extension, dotnet-service, component-library, website, app, library, other), and optional **description**.
3. Appends a new entry to `project-registry.json` with `status: "product"`.
4. If the folder already has a registry entry with a different status, the entry is promoted to `product` in place.
5. Scaffolds `CLAUDE.md` if missing, using a CieloVista-standard template.
6. Scaffolds `README.md` if missing, using the project name and description.
7. Offers follow-up actions: **Open Registry** (opens the JSON) or **Open Folder**.

## What it does NOT do

- Does not scan the folder for existing code structure or auto-detect project type.
- Does not run any build or install.
- Does not demote or archive — there is no corresponding `cvs.registry.demote` yet.
- Does not overwrite existing CLAUDE.md or README.md files.

## Registry schema

Entries written by this feature match the current `ProjectEntry` contract:

```json
{
  "name":        "JesusFamilyTree",
  "path":        "C:\\Users\\jwpmi\\Downloads\\one-electron-universe\\generated\\JesusFamilyTree",
  "type":        "website",
  "description": "Biblical genealogy visualization generated by createWebsite, promoted to product",
  "status":      "product"
}
```

The `status` field supports four values — `product`, `workbench`, `generated`, `archived` — but this command only ever writes `product`. The three-tier lifecycle model is documented in `docs/_today/CURRENT-STATUS.md`.

## Test steps

1. Open any folder in VS Code that is NOT yet registered.
2. Right-click the folder in Explorer → **Promote Folder to Product**.
3. Accept the default name, pick a type, leave description blank.
4. Open `C:\Users\jwpmi\Downloads\CieloVistaStandards\project-registry.json` and verify the new entry exists with `"status": "product"`.
5. Open the folder and verify `CLAUDE.md` and `README.md` were created.
6. Run the command a second time on the same folder — verify it does NOT duplicate the registry entry, does NOT overwrite the scaffolded docs, and reports "already in registry".
